# 系统架构说明

## 硬件架构

### 电池组结构
- **总规模**：20 个模组固定串联
- **每个模组**：4×4 电芯矩阵（共 16 个电芯）
- **电芯类型**：锂离子电芯，标称电压 3.6V，容量 22Ah
- **总配置**：320 个电芯（20×16）


### 电气参数
- **总电压范围**：300‑450V（由串联模组数决定）
- **最大电流**：单个电芯 3C（150A），模组内并联可倍增

## 软件架构

### 分层设计

```
┌─────────────────────────────────────────┐
│            Agent 决策层                  │  ← 10 分钟周期
│  • 策略选择（基于导航+电池统计）         │
│  • 输出策略代号 + 参数 (k)               │
├─────────────────────────────────────────┤
│           电芯挑选层                     │  ← 1 秒周期
│  • 执行挑选算法（按策略权重评分）        │
│  • 输出每个模组的选中电芯列表            │
├─────────────────────────────────────────┤
│           拓扑生成层                     │  ← 10 毫秒周期
│  • 根据策略生成开关矩阵指令              │
│  • 校验安全边界（电压/电流/温度）        │
├─────────────────────────────────────────┤
```

### 数据流

1. **输入**：
   - 导航信息（10 分钟片段，JSON）
   - 电池状态（所有电芯的 SOC、电压、温度、内阻、健康度）
   - 环境数据（环境温度、海拔、时间）

2. **处理**：
   ```
   导航信息 + 电池状态
       ↓
   Agent 决策（选择策略）
       ↓
   电芯挑选（按策略权重评分，选 top‑k）
       ↓
   拓扑生成（串并联映射）
       ↓
   安全校验（电压/电流/温度边界）
   ```

3. **输出**：
   - 状态反馈（当前拓扑、电芯利用率、警告信息）
   - 日志记录（策略切换、故障事件、性能指标）

## Agent 决策模块

### 输入特征提取

从导航信息中提取的关键特征：

| 特征名 | 提取方法 | 用途 |
|--------|----------|------|
| 功率需求 | `P_req = (gain_m × 车辆质量 × g) / (duration_min × 60)` | 判断是否需要高功率 |
| 能量需求 | `E_req = dist_km × 能耗系数` | 判断是否需要高能量 |
| 热负荷 | `temp + (gain_m × 0.1)` | 预测温度上升 |
| 路况复杂度 | `road_type_score + traffic_score` | 决定策略保守程度 |

从电池状态中提取的统计特征：

| 特征名 | 计算方法 | 用途 |
|--------|----------|------|
| 平均 SOC | 所有电芯 SOC 均值 | 判断是否触发均衡 |
| SOC 标准差 | 电芯间 SOC 差异 | 评估均衡需求 |
| 最高温度 | 所有电芯温度最大值 | 触发热管理 |
| 平均内阻 | 所有电芯内阻均值 | 评估功率能力 |
| 健康度分布 | 健康度 < 0.8 的电芯比例 | 决定经济性策略 |

### 决策逻辑

基于规则的决策树（可替换为 ML 模型）：

```
if 故障电芯 detected:
    选择 FT 策略
elif 最高温度 > 45°C:
    选择 TM 策略
elif gain_m > 50 或 speed_delta > 20:
    选择 HP 策略
elif dist_km > 8 且 speed > 50:
    选择 HE 策略
elif 平均 SOC < 30% 且 duration_min < 5:
    选择 BL 策略
elif 充电枪 connected:
    选择 RC 策略
elif speed < 30 且 traffic = "拥堵":
    选择 EC 策略
else:
    保持当前策略或选择 SA
```

## 电芯挑选算法

### 评分函数细节

电芯评分基于多目标加权，首先对原始数据进行归一化：

```python
def normalize(value, min_val, max_val, reverse=False):
    """归一化到 [0, 1] 区间"""
    clipped = max(min(value, max_val), min_val)
    scaled = (clipped - min_val) / (max_val - min_val)
    return 1 - scaled if reverse else scaled

# 各指标归一化范围
NORM_RANGES = {
    "soc": (0.2, 1.0),           # SOC 低于 20% 视为危险
    "temp": (15, 45),            # 15-45°C 为工作范围，25°C 最佳
    "internal_resistance": (0.02, 0.05),  # 内阻越小越好
    "health": (0.5, 1.0),        # 健康度低于 50% 视为失效
    "cycle_count": (0, 1000)     # 循环次数越少越好
}
```

### 模组内挑选流程

1. **数据准备**：读取模组内 16 个电芯的状态数据
2. **评分计算**：按策略权重计算每个电芯的加权得分
3. **排序筛选**：按得分降序排列，选取前 k 个电芯
4. **多样性约束**：避免选中电芯全部来自同一行/列，分散热应力
5. **输出**：选中电芯的 ID 列表及其得分

### 跨模组协调

为了均衡模组间差异，引入模组级调整因子：

```python
# 模组调整因子：让 SOC 高的模组多放电，SOC 低的模组休息
module_factor = (module_avg_soc - global_avg_soc) * 0.5 + 1.0
# 调整后的 k：SOC 高的模组多选电芯，SOC 低的模组少选
adjusted_k = int(base_k * module_factor)
```

## 拓扑生成器

### 串并联映射规则

根据策略决定模组内拓扑：

| 策略 | 模组内连接 | 模组间连接 | 示例（k=3） |
|------|------------|------------|-------------|
| HP | 选中电芯并联 | 串联 | 3 个电芯并联，形成低阻路径 |
| HE | 选中电芯串联 | 串联 | 3 个电芯串联，提高电压 |
| BL | 高 SOC 电芯串联放电 | 串联 | SOC 最高的 2 个电芯串联放电 |
| TM | 发热电芯旁路，其余并联 | 串联 | 旁路温度最高的 1 个电芯，其余 15 个并联 |
| FT | 故障电芯旁路，重组拓扑 | 串联 | 旁路故障电芯，剩余电芯重新串并联 |
| RC | 低 SOC 电芯并联充电 | 串联 | SOC 最低的 4 个电芯并联接充电器 |
| EC | 年轻电芯串联，老化电芯并联辅助 | 串联 | 循环次数少的 3 个电芯串联，老化电芯并联 |
| SA | 所有电芯串联 | 串联 | 16 个电芯全部串联 |

### 开关指令编码

每个开关状态用 1 位表示（0=断开，1=闭合），一个模组的开关矩阵需要：

- **行内开关**：4 行 × 3 个列间开关 = 12 个开关
- **列间开关**：4 列 × 3 个行间开关 = 12 个开关
- **旁路开关**：16 个电芯各 1 个旁路开关 = 16 个开关
- **总开关数**：40 个开关/模组，全系统 800 个开关

指令格式（CAN 报文）：
```
ID: 0x200 + module_id
Data[0-4]: 开关状态位图（40 位，5 字节）
Data[5]:  校验和
Data[6-7]: 保留
```

## 安全监控

### 实时监控项

| 监控项 | 采样频率 | 阈值 | 应急动作 |
|--------|----------|------|----------|
| 电芯电压 | 100Hz | <2.8V 或 >4.2V | 立即旁路该电芯 |
| 电芯温度 | 10Hz | >50°C | 旁路并触发 TM 策略 |
| 总电压 | 100Hz | <300V 或 >450V | 切换 SA 策略，限制功率 |
| 总电流 | 100Hz | >300A（2C） | 降低功率需求，切换 HP→HE |
| 开关温度 | 1Hz | >80°C | 降低开关频率，减少负载 |
| 通信延迟 | 1Hz | >100ms | 切换 SA 策略，报警 |

### 故障处理流程

1. **检测**：传感器数据异常或超出阈值
2. **定位**：确定故障电芯/模组/开关
3. **隔离**：立即旁路故障元件
4. **重组**：重新计算拓扑，确保功能降级而非失效
5. **记录**：存储故障信息，更新健康度
6. **报警**：通知驾驶员/运维人员

## 性能指标

### 实时指标
- **决策延迟**：从接收到导航信息到输出策略的时间（目标 < 500ms）
- **挑选延迟**：从策略确定到选中电芯列表的时间（目标 < 100ms）
- **拓扑生成延迟**：从选中列表到开关指令的时间（目标 < 50ms）
- **开关切换时间**：从指令下达到拓扑就绪的时间（目标 < 10ms）

### 长期指标
- **容量利用率**：月平均放出能量 / 理论总能量（目标 > 95%）
- **均衡度**：放电结束时 SOC 标准差（目标 < 2%）
- **寿命延长**：与固定拓扑相比的循环寿命提升比例（目标 > 20%）
- **故障率**：每月故障电芯数量 / 总电芯数（目标 < 0.1%）
